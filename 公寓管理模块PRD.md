## 1. 模块概述

### 1.1 模块定位
公寓管理模块是公寓管理系统的核心基础模块,负责管理整个系统中的公寓信息,包括公寓的基本信息、楼栋信息、房间信息的统一管理和维护。

### 1.2 业务目标
- 实现公寓信息的集中化管理
- 支持多城市、多区域公寓的统一运营
- 为租户管理、订单管理、账单管理等下游模块提供基础数据支撑
- 提升公寓信息的查询效率和准确性

### 1.3 目标用户
- 公寓运营管理人员
- 系统管理员
- 财务人员(查看公寓基础信息)

## 3. 功能详细说明

### 3.1 公寓列表

#### 3.1.1 功能描述
展示所有公寓信息列表,支持多条件筛选、分页展示、新增、编辑、删除等操作。

#### 3.1.2 页面入口
左侧导航栏 → 公寓管理 → 公寓列表

#### 3.1.3 搜索筛选功能

| 字段名称 | 字段类型 | 是否必填 | 说明 | 校验规则 |
|---------|---------|---------|------|---------|
| 公寓名称 | 文本输入框 | 否 | 支持模糊查询 | 最大长度50字符 |
| 城市 | 文本输入框 | 否 | 支持模糊查询 | 最大长度20字符 |
| 状态 | 下拉选择 | 否 | 公寓运营状态 | 选项:全部/启用/停用 |

**操作按钮**:
- 搜索按钮:根据筛选条件查询公寓列表
- 重置按钮:清空所有筛选条件,恢复默认状态

#### 3.1.4 数据表格

| 列名 | 字段名 | 数据类型 | 说明 | 显示规则 |
|------|--------|---------|------|---------|
| ID | apartment_id | bigint | 公寓唯一标识 | 自动生成,自增 |
| 公寓名称 | name | varchar(100) | 公寓名称 | - |
| 公寓编码 | code | varchar(50) | 公寓内部编码 | 唯一性校验 |
| 地址 | address | varchar(500) | 公寓详细地址 | - |
| 房间总数 | room_count | int | 该公寓下房间总数 | 自动统计 |
| 房间状态 | room_status | varchar(100) | 房间状态汇总 | 显示"空置X/已租Y/维修中Z" |
| 联系电话 | phone | varchar(20) | 管理方联系电话 | 手机号或座机格式 |
| 状态 | status | tinyint | 公寓运营状态 | 0=停用(灰色标签) 1=启用(绿色标签) |
| 创建时间 | created_at | datetime | 记录创建时间 | 格式:YYYY-MM-DD HH:mm:ss |
| 操作 | - | - | 操作按钮列 | 编辑/删除/启用停用 |

#### 3.1.5 操作功能

**新增公寓**:
- 入口:页面顶部"+新增公寓"按钮
- 操作:点击后跳转至新增公寓表单页面
- 权限:需要公寓管理权限

**编辑公寓**:
- 入口:表格操作列"编辑"按钮
- 操作:点击后弹出编辑表单或在当前页面编辑
- 权限:需要公寓编辑权限

**删除公寓**:
- 入口:表格操作列"删除"按钮
- 操作:点击后弹出二次确认框,确认后删除
- 校验:
  - 若公寓下存在楼栋或房间,不允许删除
  - 若公寓下存在租户或订单,不允许删除
- 权限:需要公寓删除权限

**启用/停用**:
- 入口:表格操作列"启用"或"停用"按钮
- 操作:切换公寓状态
- 规则:
  - 停用后,该公寓下的房间无法创建新订单
  - 已生效的订单不受影响
- 权限:需要公寓编辑权限

#### 3.1.6 分页功能

| 控件 | 说明 |
|------|------|
| 每页条数 | 可选择10/20/50/100条/页 |
| 页码导航 | 支持上一页、下一页、页码跳转 |
| 总记录数 | 显示"共X条" |
| 跳转输入框 | 支持手动输入页码跳转 |

### 3.2 公寓新增/编辑

#### 3.2.1 功能描述
新增或编辑公寓基础信息。

#### 3.2.2 表单字段

| 字段名称 | 字段类型 | 是否必填 | 说明 | 校验规则 |
|---------|---------|---------|------|---------|
| 公寓名称 | 文本输入框 | 是 | 公寓名称 | 1-100字符,唯一性校验 |
| 公寓编码 | 文本输入框 | 是 | 公寓内部编码 | 1-50字符,唯一性校验 |
| 城市 | 文本输入框 | 是 | 所在城市 | 1-20字符 |
| 详细地址 | 文本输入框 | 是 | 公寓详细地址 | 1-500字符 |
| 联系电话 | 文本输入框 | 是 | 管理方联系电话 | 手机号或座机格式校验 |
| 负责人 | 文本输入框 | 否 | 公寓负责人姓名 | 1-50字符 |
| 负责人电话 | 文本输入框 | 否 | 负责人电话 | 手机号格式校验 |
| 状态 | 单选框 | 是 | 运营状态 | 默认为"启用" |
| 备注 | 文本域 | 否 | 备注信息 | 最大500字符 |

#### 3.2.3 表单校验规则
- 所有必填字段不能为空
- 公寓名称和编码在系统中唯一
- 联系电话格式校验:
  - 手机号:1[3-9][0-9]{9}
  - 座机:\d{3,4}-\d{7,8}
- 编辑时,公寓名称和编码不能修改

#### 3.2.4 操作按钮
- 保存:保存表单数据
- 取消:关闭表单,返回列表页

### 3.3 楼栋管理

#### 3.3.1 功能描述
管理各公寓下的楼栋信息,包括楼栋的新增、编辑、删除和查询。

#### 3.3.2 页面入口
左侧导航栏 → 公寓管理 → 楼栋管理

#### 3.3.3 搜索筛选功能

| 字段名称 | 字段类型 | 是否必填 | 说明 |
|---------|---------|---------|------|
| 所属公寓 | 下拉选择 | 否 | 选择公寓,级联显示楼栋 |
| 楼栋名称 | 文本输入框 | 否 | 支持模糊查询 |
| 状态 | 下拉选择 | 否 | 启用/停用 |

#### 3.3.4 数据表格

| 列名 | 字段名 | 数据类型 | 说明 |
|------|--------|---------|------|
| ID | building_id | bigint | 楼栋唯一标识 |
| 所属公寓 | apartment_name | varchar | 公寓名称 |
| 楼栋名称 | name | varchar(100) | 楼栋名称 |
| 楼栋编码 | code | varchar(50) | 楼栋内部编码 |
| 楼层数 | floor_count | int | 楼层数量 |
| 房间数 | room_count | int | 该楼栋下房间总数 |
| 状态 | status | tinyint | 0=停用 1=启用 |
| 创建时间 | created_at | datetime | 创建时间 |
| 操作 | - | - | 编辑/删除 |

#### 3.3.5 业务规则
- 楼栋必须归属于某个公寓
- 删除楼栋前需校验楼栋下是否存在房间
- 停用楼栋后,该楼栋下房间无法创建新订单

---

### 3.4 房间管理

#### 3.4.1 功能描述
管理各楼栋下的房间信息,包括房间的新增、编辑、删除和状态管理。

#### 3.4.2 页面入口
左侧导航栏 → 公寓管理 → 房间管理

#### 3.4.3 搜索筛选功能

| 字段名称 | 字段类型 | 是否必填 | 说明 |
|---------|---------|---------|------|
| 所属公寓 | 下拉选择 | 否 | 级联选择楼栋 |
| 所属楼栋 | 下拉选择 | 否 | 根据公寓联动 |
| 房间号 | 文本输入框 | 否 | 支持模糊查询 |
| 房间状态 | 下拉选择 | 否 | 空置/已租/维修中/预订中 |

#### 3.4.4 数据表格

| 列名 | 字段名 | 数据类型 | 说明 |
|------|--------|---------|------|
| ID | room_id | bigint | 房间唯一标识 |
| 所属公寓 | apartment_name | varchar | 公寓名称 |
| 所属楼栋 | building_name | varchar | 楼栋名称 |
| 楼层 | floor | int | 所在楼层 |
| 房间号 | room_number | varchar(20) | 房间编号 |
| 房间类型 | room_type | varchar(50) | 单间/一室一厅/两室一厅等 |
| 面积 | area | decimal | 房间面积(平方米) |
| 租金 | rent | decimal | 月租金(元) |
| 房间状态 | status | tinyint | 0=空置 1=已租 2=维修中 3=预订中 |
| 创建时间 | created_at | datetime | 创建时间 |
| 操作 | - | - | 编辑/删除/查看详情 |

#### 3.4.5 房间状态说明
- **空置**:房间当前无人租住,可接受预订
- **已租**:房间已签约,租期内
- **维修中**:房间处于维修状态,无法出租
- **预订中**:房间已被预订,等待签约

#### 3.4.6 业务规则
- 房间必须归属于某个楼栋
- 同一楼栋内房间号不能重复
- 删除房间前需校验房间是否存在未完成的订单
- 房间状态变更需记录操作日志

## 5. 接口设计

### 5.1 公寓相关接口

#### 5.1.1 查询公寓列表
- **接口路径**: GET /api/apartment/list
- **请求参数**:
  ```json
  {
    "name": "公寓名称",
    "city": "城市",
    "status": 1,
    "page": 1,
    "pageSize": 10
  }
  ```
- **响应数据**:
  ```json
  {
    "code": 200,
    "message": "success",
    "data": {
      "list": [
        {
          "apartment_id": 1,
          "name": "阳光公寓",
          "code": "AP001",
          "city": "北京",
          "address": "朝阳区建国路88号",
          "room_count": 120,
          "room_status": "空置50/已租70",
          "phone": "13800138000",
          "status": 1,
          "created_at": "2026-03-18 10:00:00"
        }
      ],
      "total": 1,
      "page": 1,
      "pageSize": 10
    }
  }
  ```

#### 5.1.2 新增公寓
- **接口路径**: POST /api/apartment/add
- **请求参数**:
  ```json
  {
    "name": "阳光公寓",
    "code": "AP001",
    "city": "北京",
    "address": "朝阳区建国路88号",
    "phone": "13800138000",
    "manager_name": "张三",
    "manager_phone": "13900139000",
    "status": 1,
    "remark": "新开公寓"
  }
  ```
- **响应数据**:
  ```json
  {
    "code": 200,
    "message": "新增成功",
    "data": {
      "apartment_id": 1
    }
  }
  ```

#### 5.1.3 编辑公寓
- **接口路径**: PUT /api/apartment/update
- **请求参数**:
  ```json
  {
    "apartment_id": 1,
    "name": "阳光公寓",
    "city": "北京",
    "address": "朝阳区建国路99号",
    "phone": "13800138001",
    "status": 1
  }
  ```

#### 5.1.4 删除公寓
- **接口路径**: DELETE /api/apartment/delete/{apartment_id}
- **响应数据**:
  ```json
  {
    "code": 200,
    "message": "删除成功"
  }
  ```

#### 5.1.5 启用/停用公寓
- **接口路径**: PUT /api/apartment/status
- **请求参数**:
  ```json
  {
    "apartment_id": 1,
    "status": 0
  }
  ```

## 6. 非功能性需求

### 6.1 性能要求
- 公寓列表查询响应时间 < 500ms
- 支持并发查询用户数 > 100
- 数据库查询需建立合适索引

### 6.2 安全要求
- 所有接口需进行身份认证和权限校验
- 敏感信息(如联系电话)需要脱敏处理
- 操作日志需记录用户ID、操作时间、操作内容

### 6.3 兼容性要求
- 支持主流浏览器(Chrome、Firefox、Edge、Safari)
- 响应式设计,支持1920x1080及以上分辨率
- 移动端适配(可选)

### 6.4 可用性要求
- 系统可用性 >= 99.5%
- 数据备份:每日自动备份
- 异常处理:友好的错误提示信息

## 8. 异常处理

### 8.1 异常场景

| 异常场景 | 处理方式 | 提示信息 |
|---------|---------|---------|
| 公寓名称重复 | 不允许保存 | "公寓名称已存在,请重新输入" |
| 公寓编码重复 | 不允许保存 | "公寓编码已存在,请重新输入" |
| 删除公寓时存在关联数据 | 不允许删除 | "该公寓下存在楼栋或房间,无法删除" |
| 联系电话格式错误 | 不允许保存 | "联系电话格式不正确" |
| 网络请求失败 | 显示错误提示 | "网络请求失败,请稍后重试" |
